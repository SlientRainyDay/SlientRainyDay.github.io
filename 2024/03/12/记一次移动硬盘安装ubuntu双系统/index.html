使用移动硬盘安装ubuntu双系统

前言

因为毕设需要配置强化深度学习环境，自动驾驶后面用到ROS和Autoware的时候还用Windows可能会出各种兼容问题，后面其他的一些工作也需要高强度使用ubuntu，考虑到ubuntu虚拟机不能使用GPU的问题，故使用移动硬盘配置ubuntu双系统。看了很多老教程也是踩了不少的坑，故记录一下，以后可能还用得到。

准备：

笔者所用笔记本为拯救者Y9000P2022款，因为在后面bios设置引导U盘启动时遇到了和网上其他版本电脑引导方式的诸多差异问题，所以在后面使用引导盘启动时与网上其他博客有较大不同。

一个移动硬盘

一个U盘作为系统引导盘

rufus(百度搜索下载，使用试用版即可) 将U盘制作为引导盘 Rufus 的官网： https://rufus.akeo.ie/

DiskGenius 给硬盘分区

步骤

一.下载镜像

到Ubuntu官网找到自己想要的版本，比如20.04.6版本，下载对应的 ubuntu-20.04.6-desktop-amd64.iso 文件。

二.制作U盘引导盘

注意，此处因为不同软件制作的引导盘安装命令不同，有一些可能会出错，强烈建议使用rufus制作。

选择下载好的镜像，直接点击开始制作即可。

三.移动硬盘分区

笔者选择先在windows下建立好硬盘的ubuntu分区，这样在安装ubuntu时只需要选择对应区域就可以直接安装。

只有 转换分区表类型为GUID模式，且硬盘的最开始位置是ESP分区(EFI格式) 才能保证移动硬盘上安装的Ubuntu系统插在不同的电脑上都能运行。

打开DiskGenius，然后在左侧栏选择将要使用的硬盘，右键→转换分区表类型为GUID模式，转换完成后如图所示。

然后在上面栏的空闲分区右键建立分区。

笔者在下面建立分区时没有备注卷标，建议划分时建立卷标名称，方便安装系统时，区分不同分区。

不同博文在如下步骤差别较大，建议详细阅读加以区分。

1.建立ESP/MSR分区(引导分区)

引导分区

重点：此部分是后面安装系统时 安装启动引导器的设备 所选择的区域

2.建立新分区 — swap

该分区用于Linux系统的swap交换空间,大小建议和电脑运行内存大小保持一致。

选择文件系统类型和新分区大小，如图所示。

3.建立新分区 — ext4 /

相当于C盘，如果硬盘空间不足大于10G即可。

4.建立新分区 — ext4 /home

逻辑分区，用户存储数据用，根据存储需求应尽可能大。

5.剩余空间作NTFS格式分区

建立新分区，选择NTFS格式，大小选择剩下的所有空间，当然也可以不剩下全部分配掉。

该空间windows和ubuntu都能使用(对于windows系统来说，相当于是剩下了剩余空间当做存储空间)。

四/五项建立完成后点击左上角保存更改，

全部完成后如图所示。

四.使用引导盘安装系统

笔者电脑为联想拯救者Y9000P，bios版本比较新，在开机时直接按F12就能直接进入引导系统。

步骤就是正常安装系统，需要注意的是 安装类型选择其他选项 和 安装启动引导器的设备选择ESP0分区

关键步骤有三步：

1.开机时按F12，选择引导U盘。(不同的引导盘制作软件可能制作出来的引导盘名称不同)

2.选择将要安装的*ubuntu系统。

3.选择对应区域(关键部分)

点击两个继续。

此部分可参考新手安装 Ubuntu 操作系统步骤教程

完整步骤如下：

安装Ubuntu -> 选择语言 -> 正常安装 -> 安装类型为“其他选项” -> 安装启动引导器的设备(一定要选择 /boot 对应的分区ESP0, 不然启动不了) -> 选择地区 -> 用户名、密码设置 -> 等待安装完成
这部分的具体过程(图文详细)可以参考以下两个教程：

  • 在移动硬盘上安装Ubuntu20.04教程
  • 新手安装 Ubuntu 操作系统步骤教程
